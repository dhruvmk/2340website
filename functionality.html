Functionality

  • Welcome Screen: A friendly entry point to the app.
  • User Authentication: A login page for existing users and a streamlined account creation process for newcomers using Firebase's authentication system.
  • Initial Layout Setup: Introduction of a splash page and a user-friendly navigation bar, guiding users to key features such as Meal Input, Recipe Management, Ingredient Tracking, and Shopping List Creation.
  • Meal Logging Feature: Users can effortlessly input meals, specifying their names and estimated calorie counts, with the data securely stored in Firebase.
  • Personalized Calorie Goals: Empowering users to input personal details for the calculation of custom daily calorie targets.
  • Calorie Intake Display: Providing users with real-time updates on their current daily calorie consumption.
  • Visualizing Data: Presents calorie data in an intuitive manner using data visualization.
  • Pantry and Cookbook Databases: Establishing a dedicated Pantry Database for ingredient tracking and a Cookbook Database for storing recipes.
  • Recipe Screen Enhancements: Enabling users to add new recipes and peruse existing ones, with visual cues indicating ingredient availability.
  • Shopping Screen Enhancements: Implementing a system to prompt users to add missing ingredients to their shopping lists when preparing recipes.
  • Ingredient Management Interface: Allowing users to conveniently add, update, or remove ingredients from their pantry.
  • Recipe-Pantry Synchronization: Automating ingredient deduction from the pantry when a recipe is selected.
  • Recipe-Shopping List Synchronization: Enabling users to effortlessly add missing ingredients from selected recipes to their shopping lists.
  • Shopping List Management: Developing a robust database and user interface to manage added items and suggest further purchases based on pantry inventory.
